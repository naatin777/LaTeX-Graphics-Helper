## How to read this document

観測と判断を分ける。

- `Evidence gap`: contract、oracle、runtime、platform、package、CI接続が不足または未確認
- `Test architecture issue`: current runner/helper/fileが責務を混在させ、failure diagnosisやownershipを難しくする
- `Production architecture issue`: source側のboundaryやdependency ownershipの課題。今回修正しない
- `confirmed`: code/config/workflowから直接確認
- `likely`: current codeとtestから高い確度で推定
- `uncertain`: testを動かす環境またはimplementation boundaryの追加確認が必要
- `unknown`: inventory完了後も、branch protection、未実行platform、registry acceptance、またはformal product decisionをrepository内だけでは判定できない。推測で補わない

## Production architecture issues observed but out of scope

- `LatexSnippet` owns a `vscode.SnippetString`, making a pure-looking builder Host-dependent.
- `localeMap` reads `vscode.env.language` at module scope, making message formatting environment-dependent.
- `merge_pdf_operation` and `undo_last_conversion` import command-layer record helpers, creating transitive Host dependency.
- manifest tests import `PUBLIC_COMMAND_IDS` from activation module rather than a static manifest/ID boundary.

These observations explain why runner migration cannot be decided from test imports. They are not changed in this branch; any redesign requires a separate production task and contract review.
