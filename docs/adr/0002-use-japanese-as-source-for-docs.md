# ADR-0002: 日本語を作業ドキュメントの正本にする

## ステータス

採用

## 日付

2026-06-21

## 背景

仕様や設計判断を英語で書こうとすると、英語表現に時間を使い、開発や記録が止まりやすい。

まずメンテナが正確に書ける日本語で内容を確定し、公開向けに英語が必要な文書だけを翻訳する方が、判断の正確さと開発速度を両立しやすい。

## 決定

README以外の仕様、ADR、タスク、作業メモは日本語で書く。

READMEは `README.ja.md` を意味の正本とし、`README.md` はその内容を自然な英語へ翻訳した文書とする。

## 理由

- 英語表現に気を取られて開発が止まることを防げる
- 仕様や判断を、まず日本語で正確に記録できる
- 公開向けの英語READMEは、内容の確定後にAIを使って翻訳できる
- 日本語版と英語版のどちらが正しいか迷わずに済む

## 代替案

### すべてのドキュメントを英語で書く

メリット:

- 海外の利用者やコントリビューターが読みやすい
- 翻訳による差分が発生しない

デメリット:

- 英語表現の確認に時間がかかる
- 記録の心理的な負担が増え、文書化自体を後回しにしやすい

### 日本語版と英語版を同格で管理する

メリット:

- どちらの言語からでも直接更新できる

デメリット:

- 内容が食い違ったときに正しい版を判断できない
- 両方を同時に保守する負担が増える

## 結果・影響

- 仕様や判断を日本語で素早く記録できる
- 英語圏の開発者が作業ドキュメントを読むには翻訳が必要になる
- READMEの内容に差異がある場合は `README.ja.md` を基準に修正する
- `README.md` の更新が一時的に遅れる可能性を受け入れる

## 運用ルール

- 仕様、ADR、タスク、作業メモは日本語で書く
- `README.ja.md` を意味の正本として扱う
- `README.md` は `README.ja.md` を元に自然な英語で書く
- 英語READMEに、日本語READMEや実装にない機能を追加しない
- READMEの内容を変更するときは、先に `README.ja.md` を更新する

## 見直す条件

- 日本語を読めないコントリビューターが継続的に参加するようになったとき
- 作業ドキュメントの英語公開がプロジェクト運営上必要になったとき
- READMEの翻訳差分を維持する負担が大きくなったとき
