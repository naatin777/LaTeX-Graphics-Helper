# outputPath検証仕様

## 基本方針

- 出力pathは実行環境のfilesystem規則で検証する。
- 禁止文字や禁止名を別の文字へ自動置換しない。
- 無効な出力pathは変換開始前に拒否する。
- 多言語文字、絵文字、全角英数字は、実行環境が禁止していない限り変更しない。

## 検証時点

テンプレート変数を展開し、絶対出力pathを解決した直後に検証する。無効なpathでは、変換、外部ツール、作業file作成、競合確認を開始しない。

## Windows

path componentごとに禁止文字、NUL、制御文字、予約device名、先頭末尾の半角空白、末尾のピリオドを拒否する。全角空白は拒否しない。

## POSIX

NULを拒否する。Windows専用の禁止文字や予約名は拒否しない。

## エラー

エラーには実行環境、問題のあるpath component、拒否理由を含める。自動置換や自動削除は行わない。
