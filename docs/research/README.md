# Research Notes

外部サービス、dependency、CLI、規格などを調査し、後続のタスクや設計判断で再利用する必要がある場合に記録する。

## 目的

- AIのセッションが変わっても調査結果を引き継ぐ
- 同じWeb調査を不必要に繰り返さない
- どの情報を根拠に判断したか追跡できるようにする
- 古い外部仕様を無条件に使い続けない

## 記録する内容

- 調査日
- 対象と確認したversion
- 公式情報源
- 確認できた事実
- プロジェクトへの影響
- 未確認事項
- 再確認が必要になる条件
- 関連するtask、spec、ADR

## 運用ルール

- 可能な限り公式documentation、公式repository、公式release情報を使用する
- Webページ全体を転記せず、判断を再現するために必要な事実だけを書く
- 推測と確認済みの事実を分ける
- 採用判断は`docs/adr/`、正式仕様は`docs/specs/`へ記録する
- 実装時は、調査日と現在日、採用予定versionを確認する
- 外部仕様や最新版が変わっている可能性がある場合は、記録だけを信用せず公式情報を再確認する
